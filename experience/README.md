# Experience — 开发经验捕获层

> 本目录记录**实际开发过程中**产生的踩坑、决策和原始观察。
> 与 `plugins/` `ruby/` `javascript/` 等主文档目录的区别：
>
> | | 主文档目录 | experience/ |
> |---|---|---|
> | 内容性质 | 源码结构性知识，客观，长期有效 | 实际开发行为，主观，时效性强 |
> | 写作视角 | 第三人称："该方法的行为是…" | 第一人称："我遇到了…" |
> | 写入时机 | 源码验证通过后 | 开发过程中随时 |
> | 生命周期 | 对应 Discourse 版本长期有效 | 可过时，可被升级到主文档 |

---

## 目录结构

```
experience/
├── README.md          ← 本文件
├── mistakes/          ← 踩坑记录：bug、根因、解法、教训
├── decisions/         ← 决策记录：为什么选 A 不选 B
└── inbox/             ← 原始观察暂存（7 天内处理）
```

→ 见 `standards/EXPERIENCE_GUIDE.md`（AI 必读，人类可参考）

## 升级路径

```
experience/mistakes/ 或 decisions/
        │
        │ 满足升级标准（复现 ≥2 次 / 主文档盲点 / 可泛化原则）
        │
        ▼
plugins/ ruby/ javascript/ patterns/（主文档，追加章节或反模式）
```

升级后，原 exp 文件顶部状态标注为 `🟡 已升级至主文档`，并注明目标文件路径。**原文件保留，不删除。**
